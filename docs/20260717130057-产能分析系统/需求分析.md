## 一、背景与问题

### 1.1 数据资产

| 数据源 | 文件名 | 规模 | 覆盖范围 |
|--------|--------|------|----------|
| 工时明细 | `2026-01-01~2026-06-30工时明细.xlsx` | 27,769 条，169 填报人，59 项目 | 2026-01-04 ~ 2026-06-30 |
| 完整花名册 | `完整花名册_合并整理.xlsx` | 194 人 | 组织架构、角色、人员状态全量 |

**工时明细字段**：项目名称、填报人、填报人部门、创建人、填报日期、投入工时(人天)、工作说明

**花名册字段**：姓名、工号、岗位、一级部门、二级部门、三级部门、四级部门、实际投入小组、角色、计划投入项目1-5、备注、人员类型、员工状态、岗位名称、入职日期、离职日期、临时填报说明

### 1.2 核心数据指标

| 指标 | 数值 |
|------|------|
| 总投入人天 | 16,533 人天 |
| 填报人数 | 169 人 |
| 人均投入 | ~98 人天/6个月 ≈ 月均 16.4 人天 |
| 月度均值 | ~2,755 人天/月 |
| 项目数 | 59 个 |

**关键判断**：月均 16.4 人天约占标准工作日的 75%，属于接近全职水平的工时记录，数据具有分析价值，可以近似反映真实人力资源分配。

### 1.3 数据质量发现

| 问题 | 详情 |
|------|------|
| 创建人缺失 | 37.6%（10,440 条）的创建人为空 |
| 外包人员未在册 | 13 人有时工但不在花名册（名字带 `_sz` 后缀，疑似深圳外包/借调） |
| 在册零工时 | 38 人在花名册但无工时记录（可能是管理岗/产品/实习生等不填报的群体） |
| 人员交集 | 花名册 194 人 vs 工时 169 人，交集 156 人 |
| 工时分布 | 123 人集中在 100-150 人天区间，10 人不到 20 人天，极差 20 倍+ |

### 1.4 当前痛点

- **部门管理者（TL）**：不知道团队的人力花在哪些项目上，谁投入过量、谁投入不足，只能靠 Excel 筛选自己算
- **PMO**：做资源决策时只能靠 Excel 透视表，无法交互式下钻、无法跨部门对比、无法快速发现异常
- **计划 vs 实际盲区**：花名册中"计划投入项目"与实际工时之间没有关联对比

## 二、用户画像与核心场景

### 2.1 主用户 A：部门管理者（TL / 部门经理）

| 场景 | 描述 |
|------|------|
| 月初复盘 | 选部门 + 上月，看到总投入、项目分布、人员偏离度，快速定位异常 |
| 人员管理 | 发现张三投入超出部门均值 40%，下钻看到全花在项目 Y；李四近两周零填报 |
| 资源调配 | 评估下个迭代部门人力是否够，哪些人可以从收尾项目抽出来 |

### 2.2 主用户 B：PMO / 资源决策者

| 场景 | 描述 |
|------|------|
| 季度资源盘点 | 看全局项目资源分布，识别哪些项目群吃掉了过多人力 |
| 角色瓶颈识别 | 分析角色投入结构（后端/前端/测试），预判下一个周期的人力瓶颈 |
| 外包占比评估 | 区分自有人员 vs 外部人力资源的投入占比 |

## 三、关键指标定义

| 指标 | 计算方式 | 适用维度 | 说明 |
|------|---------|----------|------|
| **投入人天** | 某维度下 `投入工时` 的 sum | 人/项目/部门/角色/月 | 基础度量 |
| **投入占比** | 子维度人天 / 父维度总人天 | 同上 | 结构分析 |
| **人均人天** | 总人天 / 填报人数 | 部门/角色 | 横向对比基准 |
| **项目资源浓度** | 项目总人天 / 项目参与人数 | 项目 | 识别是否少数人扛大项目 |
| **偏离度** | (个人人天 - 同级均值) / 同级均值 | 人 | 异常判定，阈值 50% |
| **角色投入结构** | 各角色人天 / 总人天 | 项目/部门 | PMO 资源结构分析 |
| **部门投入集中度** | Top-N 项目人天 / 部门总人天 | 部门 | 人力分散度 |

### 3.1 偏离度判定规则

- 同一部门/角色下，个人总投入偏离均值超过 **±50%**（可配置）标为异常
- ±30%~50% 标黄，±50% 以上标红

### 3.2 外包标记规则

- 名字含 `_sz` 后缀 → "外部人力资源"
- 有时工但花名册无匹配记录 → "外部人力资源（未在册）"

## 四、MVP 功能清单（Done 条件）

### 4.1 数据导入层

- [ ] **D1** 提供数据初始化脚本(`scripts/import_data.py`)，读取两份 Excel 并落库（SQLite）
- [ ] **D2** 工时明细与花名册按"姓名"关联匹配，标记人员类型：正常匹配 / 在册无工时 / 有时工不在册（外包嫌疑）
- [ ] **D3** 单位为人天，存储与展示均为人天

### 4.2 首页仪表盘

- [ ] **D4** 顶部 4 个数字卡片：总人天、填报人数、项目数、部门数，随全局筛选联动
- [ ] **D5** 月度总人天趋势折线图（1-6 月），可点击月份下钻

### 4.3 项目投入看板（PMO 视角）

- [ ] **D6** 项目投入人天排行榜——表格 + 柱状图，支持时间范围筛选（月/季/自定义）
- [ ] **D7** 点击项目下钻到人员投入明细（姓名/角色/部门/人天）
- [ ] **D8** 叠加筛选：部门、角色、人员类型（在编/外包）
- [ ] **D9** 选定项目的月度趋势折线图

### 4.4 部门/团队看板（TL 视角）

- [ ] **D10** 按部门筛选（默认二级部门），展示部门概况（总人天、人均人天、项目分布饼图）
- [ ] **D11** 人员投入排行榜（姓名/角色/总人天/偏离度标记）
- [ ] **D12** 点击某人下钻到项目投入明细 + 月度分布
- [ ] **D13** 异常高亮：偏离度 > ±50% 标红，±30%~50% 标黄

### 4.5 角色维度分析（PMO 视角）

- [ ] **D14** 按角色聚合，展示各角色总投入及部门分布
- [ ] **D15** 人力结构：正式/离职/兼岗/实习/顾问/外部 各自人天占比（饼图 + 表格）

## 五、边界与非目标（YAGNI）

| 不做 / 推迟到 V2 | 原因 |
|------------------|------|
| 权限与登录 | 母版不预置鉴权，信任环境单用户使用 |
| 实时数据同步 | 数据源是离线 Excel，对接外部系统 |
| 预算 vs 实际对比 | 计划投入项目字段为自由文本，需人工映射 |
| 产能趋势预测 | 半年数据不足以做可靠预测 |
| Excel / PDF 导出 | V2 按需迭代 |
| 个人工时填报 | 本系统只做分析，不做填报 |
| 移动端适配 | 桌面优先 |
| 多数据源版本对比 | 当前仅一份半年数据 |

## 七、数据更新流程

```
每月收到新 Excel → 覆盖替换旧文件 → 运行 pnpm import-data → 全量重新导入 → 分析结果自动更新
```

- 导入脚本：`scripts/import_data.py`
- 触发方式：`uv run --directory apps/back python scripts/import_data.py`
- 导入策略：清空旧数据 + 全新导入（全量替换）
- 启动时自动检测：若数据库为空则自动执行导入

## 八、术语表

| 术语 | 定义 |
|------|------|
| 人天 | 投入工时单位，1 人天 = 1 个人工作 1 天 |
| 填报人 | 在工时明细中记录投入的人 |
| 偏离度 | 个人投入相对于同级均值的偏差百分比 |
| 外部人力资源 | 名字含 `_sz` 或不在花名册中的填报人 |
| 二级部门 | 花名册中 `二级部门` 字段，如"应用产品部"、"协同产品部" |
| 人员类型 | 花名册中字段：在岗在编 / 在场 / 预入场 |
| 员工状态 | 花名册中字段：正式 / 离职 / 兼岗 / 实习 / 顾问 / 其他 |
