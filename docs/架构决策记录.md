# 架构决策记录(ADR)

母版里"为什么这么选"的速查表。改动母版前先看这里,别无谓推翻已定的权衡;新增重大决策往下追加一行。

## 决策速查

| # | 决策点 | 选择 | 为什么 | 取舍 / 边界 |
|---|--------|------|--------|-------------|
| 1 | HTTP 方法 | 只用 GET / POST | 心智极简, 非专家团队够用 | 更新/删除走 POST 子路径, 不用 PATCH/PUT/DELETE |
| 2 | 后端分层 | 完整三层 route→service→repository | 职责清晰、可测、可扩展 | 无数据访问的端点(如系统指标)可退化为 route→service |
| 3 | DTO | SQLModel 全家桶(Base/Table/Create/Update/Public) | 表结构与 API 契约解耦 | 每实体 5 个类, 用 Base 抽公共字段减重复 |
| 4 | 列表返回 | 分页信封 `Page[T]`(`GET ?limit&offset`) | 真实业务列表必分页; 仍只用 GET | 破坏性契约(数组→信封), 前后端须同步 |
| 5 | 定时任务 | 进程内 APScheduler, 随 lifespan 起停 | 与"单进程同源、无 nginx"一致, 零额外基建 | 多副本会重复触发; 上多副本需外部调度 / 分布式锁(不预置) |
| 6 | 定时任务访问数据 | job 经 `session_scope()` + service | 请求作用域外也守三层, 禁 job 内裸 SQL | — |
| 7 | 前端路由 | react-router 声明式(library 模式) | 多页是"任何形态"的地基; 不上 framework / Remix 模式 | — |
| 8 | SPA 深链兜底 | 后端 `SpaStaticFiles`(404→index.html) | 同源单进程下深链刷新不 404; 比 catch-all 干净(StaticFiles 挂在 `/` 时 catch-all 不命中) | 不存在的 `/assets/*` 也回 index.html(SPA 常见, 可接受) |
| 9 | 日志出口 | structlog 双出口: 控制台人读 + 文件 JSONL | 生产要机读结构化 + 人读; UTC / 固定字段 `timestamp·level·logger·message` / 结构化异常 | — |
| 10 | 日志滚动 / 留存 | 8 小时滚动、留 30 份(= 10 天)自动清理 | 覆盖 10 天窗口并自动清过期 | 由 `APP_LOGGING__*` 环境变量覆盖 |
| 11 | 配置来源 | pydantic-settings 纯 env / .env(12-factor) | Python 最主流; 各环境差异靠 env, 不建每环境文件; 便于 ConfigMap / Nacos 注入 | 放弃 YAML 声明式(曾评估 Spring 风格, 最终取 12-factor) |
| 12 | 配置热更 | `reload_settings()` seam | 预留配置中心 watcher 的接入点 | 只留 seam, 不预置具体客户端(YAGNI) |
| 13 | .env 是否入库 | **提交** `apps/back/.env`(仅无密钥默认值) | 母版给非专家: 隐藏文件 + 让人 `cp` 他们不会; 可见可改、开箱即用 | 标准做法是忽略 .env; 此为**母版特例**, .gitignore 开负例, **严禁写入真实密钥**(密钥走环境变量) |
| 14 | 质量闸门 | Ruff + mypy(strict) + pytest / Biome + tsc + Vitest, 一键 `pnpm check` | Python 无编译期, 三闸门当"编译期替身"; 覆盖率 ≥ 80% | 手动一键、不拦提交; pre-commit / CI 留未来 |
| 15 | 部署形态 | 同源单进程(`pnpm build` + `pnpm start`) | 无 nginx / 反代 / 跨域 / 多进程编排, 一台机一个端口上线 | SQLite 单写、生产单 worker; 要横向扩并发换 Postgres |
| 16 | Agent 团队 | 5 个项目级 subagent(单会话内流水线)+ `memory: project` | 最简 Agent 团队形态, 零额外基建; 越用越懂本项目 | 记忆需认真裁剪 `MEMORY.md` 并提交共享 |

## 有意不做(YAGNI, 需要时按对应 seam 接入)

鉴权(仅 `core/security.py` 占位)、CI / pre-commit(文档化为未来)、任务队列(Celery / RQ)、多租户 / 软删 / 审计字段、配置中心的具体客户端、YAML 声明式配置。
